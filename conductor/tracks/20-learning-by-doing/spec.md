# Learning-by-Doing — Incremental Multi-Goal Evolution — Specification

## Goal
Turn scrt-evolve from a one-shot `discover → generate → train` run into a
**continuous, multi-goal, learning-by-doing loop**: a local model that
incrementally evolves toward a set of declared **goals** by distilling the
**traces a frontier model produces while doing real work locally**.

The corpus is no longer a static directory — it is **generated by doing**.
As a frontier agent (Claude/etc.) works on the user's projects, it stashes
goal-relevant findings to the scrt mind palace (steered by the paired
`scrt-evolve` skill) and leaves behind transcripts. scrt-evolve harvests both,
distills them into training rows, and evolves the local model **one
eval-gated round at a time**. The expensive frontier behavior becomes a cheap,
owned, durable local artifact — "AI as an asset."

This is the product capstone the earlier tracks were building toward: the
DESIGN.md north-star **daemon**, made buildable and safe.

## The loop (one round, per goal)
```
                ┌──────────────────────────────────────────────┐
   frontier ───▶│ scrt-evolve SKILL steers palace saves         │
   model does   │  → goal-tagged stashes + raw transcripts      │
   real work    └───────────────────┬──────────────────────────┘
                                     ▼
   discover (palace_tags=<goal.tag>, palace_search=<goal.topic>)   ← already built (track 01)
                                     ▼
   generate (distill stashes + transcripts → ~100+ rows)           ← built (02) + new transcript path
                                     ▼
   CHECKPOINT  →  train (one round)  →  EVAL (probe set, per goal) ← via track 15 txn + track 10 eval
                                     ▼
                 ┌─── pass gate ───▶ KEEP: commit, advance last_good, admit rows
                 └─── regress ─────▶ ROLLBACK: restore checkpoint, discard rows
                                     ▼
   generation-improves-itself: the kept model becomes a better teacher/        ← regen, track 11
   generator for the NEXT round (the flywheel)
```
A **scheduler** runs these rounds across all goals on a cadence (or on demand),
round-robin or priority-weighted, bounded by a budget/stop-condition.

## Scope

### 1. Multi-goal config (`[[goals]]` in `evolve.toml`)
Each goal declares what to evolve toward and how its traces are captured:
- `name` — stable id (e.g. `scrt-cli-fluency`).
- `topic` — the subject (feeds `discover.palace_search`).
- `tag` — the palace tag the skill stamps on goal-relevant stashes (feeds
  `discover.palace_tags`). One goal ⇄ one tag is the contract.
- `project` — optional path scoping the corpus/transcripts to a project.
- `probe_set` — optional path to the held-out eval probes for this goal
  (consumed by track 10's `Scorer`; gates this goal's rounds).
- `weight` / `cadence` — optional scheduler hints.
Backwards compatible: no `[[goals]]` ⇒ today's single-run behavior. Goals are
**additive** `Option`/`Vec` fields (styleguide §1).

### 2. The `scrt-evolve` SKILL (pairs with `scrt-context`)
A `SKILL.md` (authored in this track) that makes a frontier agent *participate*
in the loop. It instructs the agent: **while working on a declared goal's
topic, stash relevant findings/decisions to the palace tagged with the goal's
`tag`** (`scrt … --mp-stash <name> --mp-tag <goal.tag>`). It pairs with
`scrt-context` (which already teaches palace mechanics) — scrt-context is *how*
to stash; scrt-evolve is *what to stash for, and why* (the stashes are training
signal). The skill is the human/agent-facing half of "learning by doing": the
palace fills with goal-relevant traces as a *side effect of real work*.

### 3. Trace harvesting (two signal sources)
- **Curated — goal-tagged palace stashes (primary).** Discover already filters
  by `palace_tags`/`palace_search` (built). These are high-quality, human/agent-
  curated traces. Distilled by the existing generate pipeline.
- **Raw — transcripts (firehose, lower-trust).** The frontier model's local
  work also leaves transcripts (conversation/tool-call logs). A new harvester
  captures these to `work_dir/traces/<goal>/<slug>-<date>.jsonl`, filters them
  (the scrt "capture-then-filter" rule), and distills the goal-relevant parts
  into `completion`/`instruction` rows. Lower-trust: deduped, filtered, gated
  harder than stashes; provenance stamped `gen=trace:<goal>`.

### 4. Eval-gated incremental rounds (medium rounds, keep|rollback)
Each round generates a **medium** dataset (~100+ rows, not tiny) and runs
**through track 15's transaction wrapper**: checkpoint → train one round →
evaluate against the goal's probe set (track 10 `Scorer`) → **keep** (commit +
advance `last_good` + admit rows to the corpus) on pass, **rollback** (restore
checkpoint + discard rows) on regress. No round mutates weights outside the
transaction. Catastrophe (correctness collapse / NaN) → auto-rollback +
quarantine the round's provenance + halt (track 15).

### 5. Generation-improves-itself (the flywheel)
After a kept round, the improved local model can serve as (part of) the
**teacher/generator** for the next round (regen, track 11) — its own refreshed
checkpoint hot-swapped in. Bounded by the `antagonist_ratio` decay + teacher
anchor (track 11's anti-collapse rails). Optional per goal; the API teacher
remains the safe default.

### 6. Scheduler
A bounded driver that runs rounds across goals: round-robin or weighted by
`goal.weight`, on a `cadence` or on demand, with an explicit budget/stop-
condition (no unbounded loops — styleguide §2.5). Non-interactive + resumable
(so the DESIGN daemon can drive it). The daemon may ONLY auto-evolve THROUGH
this scheduler, which itself goes THROUGH track 15.

## Constraints
- **Builds ON, does not reinvent:** discover palace-search (01, built), generate
  (02, built), Python real-model train + infer + GGUF (track 19, built), the
  eval harness `Scorer`/`StepVerdict`/`gate` (track 10), the transactional
  keep|rollback + quarantine wrapper (track 15), regen flywheel (track 11). This
  track is the **orchestration + config + skill + trace-harvest** layer over
  those, not new ML.
- **Privacy + trust:** transcripts are local and lower-trust. The harvester
  must filter (capture-then-filter), never ingest raw multi-KB transcripts
  blindly, and stamp provenance so a bad trace round is quarantinable. No
  transcript leaves the machine.
- **Additive / non-breaking:** absent `[[goals]]` = today's single run. The
  skill is opt-in. Default build stays ML-free + Python-free (heavy ML via the
  track-19 subprocess).
- **Bounded everything:** every loop (rounds, scheduler cadence, regen swaps)
  carries an explicit budget/stop-condition. No round mutates weights outside
  the track-15 transaction.

## Acceptance
- `evolve.toml` parses `[[goals]]` (name/topic/tag/project/probe_set/weight/
  cadence); absent goals reproduces single-run behavior.
- The `scrt-evolve` SKILL.md exists, pairs with `scrt-context` (cross-references
  it), and concretely instructs goal-tagged stashing; a worked example shows a
  stash tagged with a goal tag then discovered via `palace_tags`.
- A round runs end-to-end for one goal: discover(goal-tag) → generate(~100+
  rows) → checkpoint → train → eval(probe set) → keep|rollback, with the verdict
  logged to `evolution-log.jsonl`. A forced regression rolls back (state
  restored); a forced catastrophe halts + quarantines.
- The transcript harvester captures a fixture transcript, filters it, and
  produces `trace:<goal>`-stamped rows that round-trip through the dataset
  contract; gate-failing trace rows never enter the kept corpus.
- The scheduler runs ≥2 rounds across ≥2 goals, bounded by a budget, resumable
  after interruption.
- Generation-improves-itself: a kept round's model can serve as the next round's
  generator (regen), with the anti-collapse rails active.

## Dependencies
- **Built/usable now:** 01 (discover + palace-search), 02 (generate/dataset),
  19 (Python train/infer/GGUF).
- **Lane (must exist for full acceptance):** 10 (eval harness — the gate),
  15 (transactional keep|rollback + quarantine), 11 (regen flywheel, optional).
- Pairs with the external `scrt-context` skill (scrt-cli repo).

## Honest risks
- **Trace quality is unproven.** Frontier traces may be noisy or off-goal;
  the eval gate (10/15) is the backstop — a round that doesn't improve the
  probe score is rolled back, so bad traces cost time, not correctness.
- **Echo chamber via generation-improves-itself** (track 11 risk) — mitigated
  by the decaying antagonist ratio + teacher anchor + the executable gate.
- **Skill compliance is soft.** Goal-tagged stashing depends on the agent
  following the skill; the tag filter degrades gracefully (untagged stashes
  still seed if `palace_tags` is empty). Auto/hooked stashing is a future
  hardening, not v1.
- **Scope creep into "agent framework."** Kept tractable by being orchestration
  over EXISTING tracks + a markdown skill + a bounded scheduler — no new ML,
  no new model logic. Weight changes go THROUGH track 15.
