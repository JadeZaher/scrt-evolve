# scrt-evolve — Learning-by-Doing Evolution Skill

## The mental model

`scrt-context` taught you the **lens + mind palace**: search, stash, compose,
prune. `scrt-evolve` adds one idea on top: **your stashes are training data.**

A project's `evolve.toml` declares **goals** — things its owner wants a local
model to get better at. Each goal carries a `tag`. When you do real work that
advances a goal, you **stash the finding tagged with that goal's tag**. Later,
scrt-evolve's `discover` pulls exactly those goal-tagged stashes
(`palace_tags`), `generate` distills them into supervised rows, and a local
model trains on them — eval-gated, kept only if it actually improves.

So you are not just remembering for yourself. **You are writing the
curriculum.** A frontier model's expensive, high-quality work becomes a cheap,
owned local model — "learning by doing."

## When to use

- The project has an `evolve.toml` with `[[goals]]`. Read the goals: each has a
  `name`, a `topic`, and a `tag`.
- You're investigating, deciding, or solving something **inside a goal's
  topic** — and the finding would help a model do this task next time.
- You've discovered a correct command, a non-obvious mechanism, a good
  decision + its rationale, a worked example, a fix. That's curriculum.

## The core move: goal-tagged stashing

When a finding advances a goal, stash it tagged with the goal's `tag`:

```bash
# evolve.toml declares: name="scrt-cli-fluency", topic="mp-* commands", tag="scrt-cli"
scrt "mp-compose" --in src --mp-stash compose-vs-intersect \
  --mp-tag scrt-cli --mp-ttl 24h
```

The stash's **note** matters — write it as the *lesson*, not a bookmark:

```bash
scrt --mp-stash ttl-flag --mp-tag scrt-cli \
  # note: "--mp-ttl <dur> sets a stash's time-to-live (e.g. 4h, 24h);
  #        expired stashes are dropped by --mp-prune-expired."
```

A well-noted, goal-tagged stash becomes a clean qa/instruction row. A bare
bookmark becomes noise. Stash **lessons**, not links.

## What makes a good training stash

| Stash this (curriculum) | Skip this (noise) |
| :--- | :--- |
| A correct command + what it does | A file path you'll reopen |
| A mechanism + why it works | A raw multi-KB tool dump |
| A decision + its rationale | A secret / credential / token |
| A fix + the bug it solved | Something off-goal |
| A worked example end-to-end | A half-formed guess |

Tag by the **active goal**. If a finding spans two goals, tag both. If it
matches no declared goal, don't tag it for evolution (a normal `scrt-context`
stash is fine).

## How the loop uses your stashes (so you know why it matters)

```
you do real work ──▶ goal-tagged stashes (this skill)
                          │
   discover --mp-tag <goal.tag>   ← scrt-evolve pulls your goal-tagged stashes
                          ▼
   generate  ← distills your stash notes into qa/instruction rows
                          ▼
   train (real model) → EVAL → keep|rollback   ← only improvements are kept
                          ▼
   a local model that learned from your work (export to GGUF for LM Studio)
```

Your raw transcripts are also harvested (lower-trust, filtered) — but the
**goal-tagged stashes are the high-signal curriculum**. The better your notes,
the better the model.

## Pairing with `scrt-context`

- `scrt-context` = **how** to search/stash/compose/prune (the lens + palace).
- `scrt-evolve` = **what** to stash for, and **why** (goal-tagged curriculum).

Use scrt-context's mechanics; add scrt-evolve's discipline: when working a
declared goal, leave goal-tagged lessons behind. Everything in scrt-context's
"long-session memory management" still applies (TTL + tag your stashes) — here
the **tag is the goal**, and that tag is load-bearing for training.

## Hygiene (don't poison the curriculum)

- **Never stash secrets** (keys, tokens, credentials, private data). The
  palace becomes training data; a leaked secret becomes a learned secret.
- **Curate, don't firehose.** Goal-tagged stashes are the high-trust signal —
  keep them lessons, not dumps. Raw capture is the transcript harvester's job.
- **One goal ⇄ one tag.** Match the project's declared `tag` exactly, or
  discover won't find your stash.
- **Prune off-goal scratch** at session close (`scrt --mp-prune-tag scan`); keep
  the goal-tagged findings.

## Verify a stash will be picked up

```bash
# confirm your goal-tagged stashes are discoverable the way scrt-evolve will pull them
scrt --mp-list --mp-list-tag scrt-cli          # exact tag filter
scrt --mp-list --mp-find ttl --mp-list-tag scrt-cli   # search + tag, as discover composes them
```

If your stash shows up here, scrt-evolve's `discover` (with
`palace_tags=["scrt-cli"]`) will seed generation from it.
